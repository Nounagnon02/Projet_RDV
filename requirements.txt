
CAHIER DES CHARGES
Application de Gestion de Rendez-vous
Stack Technique : Laravel + React
Version 1.0 - 06/01/2026

1. PRÉSENTATION DU PROJET
1.1 Contexte
L'application vise à digitaliser et simplifier la gestion des rendez-vous pour les professionnels de services (coiffeurs, esthéticiens, consultants, thérapeutes, etc.). Elle permettra aux prestataires de gérer leur agenda en ligne et aux clients de prendre rendez-vous de manière autonome, 24h/24 et 7j/7.
1.2 Objectifs
Réduire la charge administrative liée à la prise de rendez-vous
Minimiser les erreurs de planning et les oublis
Améliorer l'expérience client avec un système de réservation simple
Optimiser le taux de remplissage des plannings
Réduire les absences avec un système de rappel automatique
1.3 Utilisateurs cibles
Prestataires de services : professionnels indépendants ou gérants de petites structures
Clients finaux : particuliers souhaitant réserver des prestations en ligne

3. SPÉCIFICATIONS TECHNIQUES
3.1 Architecture
L'application suivra une architecture client-serveur avec API RESTful :
Backend : API Laravel exposant des endpoints REST
Frontend : Application React (SPA) communicant avec l'API
Authentification : JWT (JSON Web Tokens) ou Laravel Sanctum
3.2 Stack Technique Backend (Laravel)
Framework : Laravel 10.x ou 11.x
Base de données : MySQL 8.0+ ou PostgreSQL 14+
Authentification : Laravel Sanctum pour l'API
Gestion d'emails : Laravel Mail + queues (Redis/Database)
Validation : Laravel Form Requests
ORM : Eloquent
Tâches planifiées : Laravel Scheduler (pour rappels automatiques)
3.3 Stack Technique Frontend (React)
Framework : React 18+
Build tool : Vite
Routing : React Router v6
State management : React Context API + React Query (pour le cache)
HTTP Client : Axios
UI Framework : Tailwind CSS + Headless UI ou Material-UI
Calendrier : FullCalendar ou React Big Calendar
Gestion de formulaires : React Hook Form + Yup/Zod
Notifications : React Toastify
3.4 Modèle de Données



Entité


Attributs principaux


User


id, name, email, password, phone, role (provider/client), email_verified_at, timestamps


Provider


id, user_id, business_name, description, address, city, postal_code, slug (URL), avatar, cover_image, timestamps


Service


id, provider_id, name, description, duration (minutes), price, category, is_active, image, timestamps


Availability


id, provider_id, day_of_week (0-6), start_time, end_time, is_active, timestamps


ClosedPeriod


id, provider_id, start_date, end_date, reason, timestamps


Appointment


id, provider_id, client_id, service_id, date, start_time, end_time, status (pending/confirmed/cancelled/completed), client_notes, provider_notes, timestamps


Notification


id, user_id, type (email/sms), subject, message, sent_at, read_at, timestamps


Settings


id, provider_id, slot_duration (15/30/60), cancellation_delay_hours, reminder_hours_before, timezone, timestamps
Relations principales :
User hasOne Provider (si role = provider)
Provider hasMany Services
Provider hasMany Availabilities
Provider hasMany ClosedPeriods
Provider hasMany Appointments
Client (User) hasMany Appointments
Appointment belongsTo Service, Provider, Client

4. PRINCIPAUX ENDPOINTS API
4.1 Authentification
POST /api/register - Inscription utilisateur
POST /api/login - Connexion
POST /api/logout - Déconnexion
POST /api/password/forgot - Réinitialisation mot de passe
4.2 Gestion Prestataire
GET /api/provider/profile - Récupérer profil
PUT /api/provider/profile - Mettre à jour profil
GET /api/provider/services - Liste des services
POST /api/provider/services - Créer un service
PUT /api/provider/services/{id} - Modifier un service
DELETE /api/provider/services/{id} - Supprimer un service
GET /api/provider/availabilities - Récupérer les disponibilités
POST /api/provider/availabilities - Définir les disponibilités
GET /api/provider/appointments - Liste des rendez-vous
POST /api/provider/appointments - Créer rendez-vous manuel
PUT /api/provider/appointments/{id} - Modifier rendez-vous
DELETE /api/provider/appointments/{id} - Annuler rendez-vous
GET /api/provider/statistics - Statistiques
4.3 Espace Public & Client
GET /api/booking/{slug} - Informations publiques du prestataire
GET /api/booking/{slug}/services - Services disponibles
GET /api/booking/{slug}/slots - Créneaux disponibles
POST /api/booking/{slug}/appointments - Réserver un rendez-vous
GET /api/client/appointments - Mes rendez-vous
DELETE /api/client/appointments/{id} - Annuler mon rendez-vous

5. SÉCURITÉ
Authentification basée sur tokens (Laravel Sanctum)
Hashage des mots de passe (bcrypt)
Protection CSRF pour toutes les requêtes modifiant des données
Validation stricte des entrées utilisateur
Rate limiting sur les endpoints sensibles (connexion, inscription)
Protection contre les injections SQL (Eloquent ORM)
Sanitization des sorties pour éviter les XSS
HTTPS obligatoire en production
Politique CORS stricte

13. CONCLUSION
Ce cahier des charges définit les grandes lignes d'une application de gestion de rendez-vous moderne et complète. L'utilisation du stack Laravel + React garantit une architecture robuste, scalable et maintenable.
